# Nodus — Старт разработки Rust Backend

Прочитай `CLAUDE.md` в корне проекта. Это главный источник истины.
Все решения принимай на основе него.

---

## Твоя задача

Разработать Rust backend для Nodus до состояния MVP.
MVP определён в `CLAUDE.md` — раздел "MVP — Минимально рабочий продукт".

Ты работаешь **только в `src-tauri/`**.
UI файлы (`src/`, `public/`, `index.html`) — не трогать никогда.

---

## Шаг 1 — Разведка (сделай это первым)

Перед тем как писать код:

1. Прочитай `CLAUDE.md` полностью
2. Изучи текущую структуру проекта — что уже есть в `src-tauri/`
3. Проверь `Cargo.toml` — какие зависимости уже подключены
4. Посмотри есть ли уже какой-то Rust код

Только после разведки переходи к планированию.

---

## Шаг 2 — Создай структуру для работы

Создай следующие папки и файлы в корне проекта:

```
.nodus/
├── plan.md          # план разработки с задачами
├── progress.md      # текущий статус — что сделано, что в работе
└── reports/
    ├── 001_foundation.md
    ├── 002_wasapi.md
    ├── 003_routing.md
    └── ... (по мере выполнения)
```

---

## Шаг 3 — Составь план разработки

Запиши план в `.nodus/plan.md` по следующей структуре:

```markdown
# Nodus Rust Backend — Plan

## Правила работы
- Максимум 3 параллельных агента
- После каждой задачи — отчёт в .nodus/reports/
- После каждой задачи — обновить .nodus/progress.md
- После каждой задачи — обновить Roadmap в CLAUDE.md (отметить выполненное)
- Никаких .unwrap() в продакшн коде
- Каждый модуль покрыт unit тестами

## Задачи

### TASK-001: [название]
Описание: что именно делается
Файлы: какие файлы создаются/изменяются
Зависимости: от каких задач зависит
Критерий готовности: как понять что задача выполнена
Тесты: что покрывается тестами

### TASK-002: ...
```

План должен покрывать весь MVP из `CLAUDE.md`.
Задачи должны быть атомарными — одна задача, один понятный результат.
Задачи должны иметь чёткие зависимости — что нужно сделать раньше.

## Следующая
TASK-002 — [название]

## Заблокировано
(если есть — что и почему)

---

## Шаг 5 — Приступай к выполнению

Начинай с TASK-001 и двигайся по плану.

### После каждой выполненной задачи:

1. **Отчёт** — создай файл `.nodus/reports/NNN_название.md`:
```markdown
# TASK-NNN — [название]

## Что сделано
[конкретно что реализовано]

## Созданные файлы
- path/to/file.rs — что делает

## Решения и компромиссы
[если были нетривиальные решения — объясни почему так]

## Тесты
[что покрыто, как запустить]

## Известные ограничения
[что не сделано намеренно и почему]

## Следующий шаг
TASK-NNN+1
```

2. **Обнови** `.nodus/progress.md` — текущий статус
3. **Обнови** `CLAUDE.md` — отметь выполненные пункты Roadmap
4. **Приступай** к следующей задаче

---

## Правила автономной работы

- Работай без лишних вопросов — если что-то неоднозначно,
  прими решение, зафикисруй его в отчёте и двигайся дальше
- Если встретил техническое препятствие которое блокирует задачу —
  запиши в `.nodus/progress.md` раздел "Заблокировано" с объяснением
  и переходи к следующей независимой задаче
- Документацию держи актуальной — устаревший `CLAUDE.md` хуже
  чем никакого
- Не трогай UI файлы ни при каких обстоятельствах

---

## Правила фиксации состояния — ВАЖНО

Контекстное окно может оборваться в любой момент.
Следующая сессия должна иметь возможность продолжить без потерь.

**Обновляй `.nodus/progress.md` в следующих точках:**
- Перед началом каждой новой задачи
- После создания каждого нового файла внутри длинной задачи
- При любом нетривиальном решении — сразу фиксируй
- Сразу после завершения задачи — до перехода к следующей

**Формат промежуточного статуса в `progress.md`:**
```markdown
## Текущая задача
TASK-NNN — [название]

### Что уже сделано в этой задаче
- [x] создан src-tauri/src/audio/wasapi.rs — базовая структура
- [x] создан src-tauri/src/audio/devices.rs — enumerate устройств
- [ ] src-tauri/src/audio/session.rs — в процессе

### Следующий шаг внутри задачи
Реализовать AudioSession::attach() в session.rs
```

**Отчёт пишется сразу после завершения задачи** — не после перехода
к следующей. Если сессия оборвётся между задачами — отчёт уже есть.

---

## Как продолжить после обрыва сессии

Если контекст оборвался — в новой сессии напиши:

```
Прочитай CLAUDE.md и .nodus/progress.md и .nodus/plan.md
Если progress.md не отражает актуальное состояние —
изучи src-tauri/ самостоятельно, обнови progress.md
и продолжи выполнение плана с текущей задачи.
```

Агент сам проведёт разведку и восстановит картину.
