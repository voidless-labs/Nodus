# t7 — Уведомление «Nodus не слышит систему»

**Статус:** 🟡 можно делать — мелкая, backend + событие + текст в UI
**Ждёт:** ничего
**Кто делает:** backend (проверка + событие), ui-bridge (показ)

## Суть простыми словами
Чтобы Nodus перехватывал звук всех приложений, в настройках Windows устройством
вывода по умолчанию должна стоять наша колонка «Динамики (Nodus)». Если стоит
что-то другое — Nodus звук системы не слышит. Нужно это заметить и спокойно
объяснить пользователю, что сделать (а в идеале — кнопка «исправить»).

## Критерий готовности
- [ ] Nodus определяет, что устройство вывода по умолчанию ≠ колонка Nodus
- [ ] В UI появляется ненавязчивое уведомление с понятной инструкцией
- [ ] Уведомление исчезает, когда пользователь переключил вывод на колонку Nodus
- [ ] (необязательно) Кнопка «сделать колонкой по умолчанию» прямо из уведомления

## Технические заметки (для меня)
- Движок уже умеет enumerate устройств и знает default (devices.rs). Нужна
  периодическая/событийная проверка + Tauri-событие в UI (например
  `system-capture-status`).
- Детект «нашей» колонки — `virtual_device::is_nodus_virtual_name` по имени.
- Текст уведомления — продуктовый, простыми словами, без жаргона.
- Связано с продуктовой политикой t5 (ADR §2.1): одна колонка-перехватчик.
