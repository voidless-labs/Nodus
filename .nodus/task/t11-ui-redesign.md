# t11 — Редизайн UI: лёгкий, дружелюбный стиль

**Статус:** 🟡 готов к работе — делаем вместе, итеративно в коде
**Ждёт:** ничего (генераторы дизайна не справились — рисуем руками в приложении)
**Кто делает:** разработчик (направление, арт-дирекция) + Claude (вёрстка/CSS с
явного разрешения, перенос токенов, переподключение логики)

## Суть простыми словами
Текущий интерфейс ощущается как профессиональный софт (панели, капс, кокпит) —
а цель Nodus в лёгкости и понятности для обычного человека. Нужно переделать
подачу: растворить раму (убрать постоянные боковые панели, контролы — внутрь
нод), одна акцентная кнопка, мягкие карточки-ноды с живым метром и слайдером,
поповер на ребре вместо инспектора, точечная сетка холста, человеческий язык.
Функциональность НЕ меняется — меняется только внешний вид и компоновка.

Бриф со всей спецификацией: `.nodus/docs/ui-redesign-prompt.md` (изначально
писался для Claude Design; теперь это наш рабочий документ).

## Критерий готовности
- [ ] Дизайн-токены в CSS-переменных (палитра, радиусы, шрифты, тени/свечения)
- [ ] Нода-карточка: иконка + имя + живой VU-метр + слайдер; состояния
      (обычная/активная-свечение/muted/выделенная/не запущено)
- [ ] Рёбра + поповер ребра (громкость/mute/баланс) — замена бокового инспектора
- [ ] Боковые панели растворены: «+ Add» (приложения/устройства), мини-статус в углу
- [ ] Пустой холст первого запуска: призрак-подсказка + пресеты сценариев
- [ ] Онбординг виртуального устройства переписан по-человечески
- [ ] Производительность: никаких вечных анимаций в покое (урок N4200)
- [ ] Вся уже подключённая логика сохранена и переподключена к новому визуалу
      (метры, save/load, solo, pan, PTT, автодетект, виртуальные устройства)

## Технические заметки
- Сюда естественно вливаются t9 (виртуальный микрофон как сток-нода) и t7
  (уведомление «не слышу систему» — место в новом UI).
- Порядок итераций (из брифа): токены/фон/типографика → нода → рёбра+поповер →
  растворение панелей → empty state. Каждый шаг — скриншот, правки, дальше.
- Зоны: вид/CSS/layout — разработчик; перенос токенов и переподключение
  invoke/событий — Claude (по правилам проекта — с явного разрешения на правку
  визуала, которое для этой задачи уже дано принципиально).
- Контракт UI↔движок (invoke-команды, события) при редизайне НЕ меняется —
  переезжает только расположение контролов, не их вызовы.
