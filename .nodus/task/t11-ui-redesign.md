## Утверждённые решения (13.06.2026)
- **Стек: TypeScript + TSX.** Текущий UI — гиперскрипт `h()` в .jsx, без TS; слой
  отрисовки всё равно переписывается, поэтому переходим на TS заодно (типизированный
  мост к Rust ловит рассинхрон invoke/событий; чище для растущего набора нод).
  Логика НЕ выбрасывается — портируется в типизированные модули.
- **Акцент — янтарный `#F5C542`** (кнопка движка, выделение).
- **Hub-нода ДА** — опциональная нода-«пульт» (как «Stream Mix» на референсе) для
  тех, кто хочет явный микшер с настройками. Слова Splitter/Mixer в UI не вводим.
- **Иконки — реальные** (извлечение из .exe через WinAPI, backend); до готовности
  извлекателя — категорийный глиф-заглушка.
- Главный референс: `.nodus/docs/ReDesign/` (general-view, Node-design[-v2],
  graph-line, bottom bar, Engine-start-btn, scene-tab, Lovable рамка+реакция, V0 add).
- Инструкция: `.nodus/docs/ReDesign/redesign-instructions.md` (приоритет над всем).
- Работаем пошагово: каждый шаг — превью разработчику, правки, дальше.

## Суть простыми словами
Текущий интерфейс ощущается как профессиональный софт (панели, капс, кокпит) —
а цель Nodus в лёгкости и понятности для обычного человека. Нужно переделать
подачу: растворить раму (убрать постоянные боковые панели, контролы — внутрь
нод), одна акцентная кнопка, мягкие карточки-ноды с живым метром и слайдером,
поповер на ребре вместо инспектора, точечная сетка холста, человеческий язык.
Функциональность НЕ меняется — меняется только внешний вид и компоновка.

Бриф со всей спецификацией: `.nodus/docs/ui-redesign-prompt.md` (изначально
писался для Claude Design; теперь это наш рабочий документ).
