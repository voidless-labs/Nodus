# t4 — Микрофон ← движок Nodus

**Статус:** ✅ сделано 13.06.2026 — движок наполняет виртуальный микрофон, проверено на железе
**Кто делает:** агент backend (VirtualRender), Team Lead (ревью, полевой тест)

## Суть простыми словами
Когда виртуальный микрофон существует (t3), нужно подключить его к движку:
любой маршрут в графе Nodus может заканчиваться нодой «Nodus Virtual Mic», и всё,
что туда стекается (микшер: голос + музыка), попадает в Discord.

## Критерий готовности
- [x] Маршрут движка в «Nodus Virtual Mic» работает: внешнее приложение,
      выбравшее этот микрофон, слышит звук из графа ✅ 13.06.2026
      (mic-route: «Динамики (Nodus)» → движок → «Микрофон (Nodus)» → браузер записал)
- [x] Громкость/mute маршрута действуют (общий механизм атомиков с WASAPI-рендером) ✅
- [x] Закрывает вторую половину MVP-сценария из CLAUDE.md ✅
- Известное: качество звука («странный» на слух) — отдельная задача t10
  (заменить MVP-линейный ресемплер). Не блокирует: звук разборчив.

## Технические заметки (для меня)
- Новый модуль `VirtualRender` (зеркало `virtual_capture.rs`): пишет f32-кадры
  из broadcast-канала движка в capture-кольцо.
- В роутинг-граф добавить выход типа Virtual Mic (узел уже есть в UI как домен).
- Латентность: целиться в ~20–40 мс буфер, иначе эхо в голосовом чате.
