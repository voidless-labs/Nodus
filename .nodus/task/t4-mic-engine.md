# t4 — Микрофон ← движок Nodus

**Статус:** ⏸ ждёт t3
**Кто делает:** я

## Суть простыми словами
Когда виртуальный микрофон существует (t3), нужно подключить его к движку:
любой маршрут в графе Nodus может заканчиваться нодой «Nodus Virtual Mic», и всё,
что туда стекается (микшер: голос + музыка), попадает в Discord.

## Критерий готовности
- [ ] В графе Nodus маршрут «Mic → Mixer → Nodus Virtual Mic» работает:
      Discord слышит реальный микрофон + подмешанную музыку
- [ ] Громкости каждого входа микшера действуют независимо
- [ ] Это закрывает второй сценарий критерия MVP из CLAUDE.md

## Технические заметки (для меня)
- Новый модуль `VirtualRender` (зеркало `virtual_capture.rs`): пишет f32-кадры
  из broadcast-канала движка в capture-кольцо.
- В роутинг-граф добавить выход типа Virtual Mic (узел уже есть в UI как домен).
- Латентность: целиться в ~20–40 мс буфер, иначе эхо в голосовом чате.
