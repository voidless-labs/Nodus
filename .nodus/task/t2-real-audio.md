## Суть простыми словами
Сейчас виртуальная колонка молча выбрасывает звук. Эта задача — научить драйвер
складывать звук в общую память, а Nodus — забирать его оттуда и направлять по нодам.
После неё сценарий «Spotify играет в Nodus Virtual Speaker → слышно в наушниках
через граф Nodus» работает целиком.

## Критерий готовности
- [ ] Spotify/браузер играет в «Nodus Virtual Speaker» → звук слышен на реальном
      выходе, выбранном в Nodus
- [ ] Громкость/mute маршрута в Nodus действуют на этот звук
- [ ] Ничего не падает при старте/остановке потока и перезапуске Nodus

## Сделано 11.06.2026
- Драйвер: `ring.h/ring.cpp` — именованная секция `Global\NodusRing-0` с явным
  DACL (SYSTEM/Admins — полный, Everyone — чтение), маппинг `MmMapViewInSystemSpace`,
  заголовок v2 (64-битные счётчики), Magic пишется последним с барьером.
- Драйвер: поток копирования в `minwavertstream.cpp` — системный поток на
  PASSIVE_LEVEL (НЕ DPC), каждые 10 мс копирует «проигранные» байты из циклического
  буфера WaveRT в кольцо (RtlCopyMemory сегментами). Teardown: событие → join потока
  → только потом освобождение буфера. Арбитраж single-writer между стримами.
- Rust: `virtual_capture.rs` переписан под v2 — новое имя секции, 64-битные
  счётчики, конвертация 16-бит PCM → f32, ресинк при отставании читателя.
  Тест-замок на layout (offset_of) + C_ASSERT в драйвере с двух сторон контракта.
- `cargo test`: 37 passed. Драйвер: проверка компиляции — CI.

## Технические заметки (для меня)
- Кольцевой буфер из старого драйвера возвращаем правильно:
  `MmMapViewInSystemSpace` вместо process-context маппинга (иначе BSOD из DPC);
  явный security descriptor (читаемо обычным пользователем);
  копирование не в DPC, а в системном потоке на PASSIVE_LEVEL.
- Имя секции сразу параметрическое: `Global\NodusRing-<id>` (задел под t5),
  для единственного устройства id = 0.
- 64-битные счётчики байт (старый ULONG переполнялся через ~3 часа).
- Rust-сторона уже готова: `src-tauri/src/audio/virtual_capture.rs` +
  `Backend::Virtual` в движке — поменять только имя секции и layout заголовка.
- Копирование блоками (RtlCopyMemory двумя сегментами), не побайтово.
