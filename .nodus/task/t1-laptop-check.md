# t1 — Проверить драйвер на тестовом ноуте

**Статус:** ✅ сделано 11.06.2026 — устройство видно, переключения стабильны, BSOD нет
**Ждёт:** ничего
**Кто делает:** ты ставишь, я разбираю результат

## Суть простыми словами
Собранный пакет драйвера нужно поставить на тестовый ноутбук и посмотреть,
появилась ли «Nodus Virtual Speaker» в настройках звука. Звука не будет —
проверяем только, что Windows видит устройство и ничего не падает.

## Шаги
1. Скачать артефакт `nodus_audio-driver-x64-Release` со страницы успешного рана:
   https://github.com/voidless-labs/Nodus/actions/runs/27311703010
2. На ноуте включить тестовый режим (один раз): `bcdedit /set testsigning on` + перезагрузка.
3. В распакованной папке от администратора: `powershell -ExecutionPolicy Bypass -File .\install.ps1`
4. Открыть Параметры → Система → Звук → устройства вывода.

## Критерий готовности
- [x] «Nodus Virtual Speaker» видна в списке устройств вывода ✅ 11.06.2026
      (в списке отображается как «Динамики (Nodus Virtual Audio)»)
- [x] Система не упала (нет BSOD) при установке и при выборе устройства ✅ 11.06.2026
      (~2 мин музыки на виртуальном устройстве, многократное переключение
      виртуальные ↔ Realtek — стабильно)
- [ ] (необязательно) В DebugView видны строки `Nodus: StartDevice ... status=0x00000000` — не проверяли, не понадобилось

## Заметки по ходу проверки
- install.ps1 падал с ParserError на русской локали: длинные тире (em dash, U+2014)
  в UTF-8 без BOM читались PowerShell 5.1 как смарт-кавычки cp1251. Исправлено
  (все .ps1 теперь чистый ASCII), пользователь патчил скрипт на ноуте one-liner'ом.

## Если не получилось
Снять лог DebugView / фото синего экрана (код + `nodus_audio.sys+смещение`) — этого
мне хватит для разбора: PDB для симвализации лежит в том же артефакте.
