# t3 — Виртуальный микрофон

**Статус:** ⏸ ждёт t1 (фундамент подтверждён) и t2 (отработанный механизм колец)
**Кто делает:** я

## Суть простыми словами
Сейчас у нас только виртуальная колонка (звук *в* Nodus). Микрофон — обратное
направление: Nodus отдаёт звук, а Discord/TeamSpeak видят «Nodus Virtual Mic»
в списке микрофонов и читают из него. Без этого не работает половина MVP
(сценарий «музыка + голос в Discord»).

## Критерий готовности
- [ ] «Nodus Virtual Mic» виден в Windows в списке устройств записи
- [ ] Discord может выбрать его как микрофон и слышит то, что отдаёт Nodus
- [ ] Ничего не падает при подключении/отключении приложений к микрофону

## Технические заметки (для меня)
- Вторая пара wave+topology в том же драйвере: capture-вариант,
  connector-пин `KSNODETYPE_MICROPHONE` вместо `KSNODETYPE_SPEAKER`.
- Поток данных зеркальный t2: отдельное кольцо `Global\NodusRing-mic-<id>`,
  Nodus пишет, драйвер раздаёт приложениям через WaveRT-буфер capture-стрима.
- Лимит subdevice в `PcAddAdapterDevice` поднять (сейчас 2, станет 4+).
- Референс — capture-путь в SYSVAD (microphone topology + wave capture stream).
