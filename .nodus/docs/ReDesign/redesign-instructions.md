# Nodus — UI Redesign Instructions

> Этот файл является главной инструкцией для Claude Code при работе над редизайном UI приложения Nodus.
> При любом конфликте между этим файлом и другими документами — этот файл имеет приоритет.

---

## Что такое Nodus

Nodus — десктопное приложение (Tauri + React) для визуального аудио-роутинга на Windows.
Пользователь соединяет источники звука (Spotify, Discord, игра, микрофон) с выходами (наушники, колонки, OBS, виртуальный микрофон) стрелками на бесконечном холсте.

**Аудитория:** стримеры-новички, геймеры, подкастеры. НЕ звукорежиссёры.
**Платформа:** Windows, только тёмная тема, минимальное окно 1280×800.

---

## Философия редизайна — читай перед любой задачей

Текущий интерфейс выглядит как «кокпит самолёта» — тяжёлый, профессиональный. Цель — **лёгкость**.

Ключевые принципы:
1. **Никаких постоянных боковых панелей.** Холст — это всё. Library, Inspector, Quick Controls переезжают внутрь нод или становятся плавающими попоперами.
2. **Контрол там, куда смотришь.** Громкость — слайдер прямо на ноде. Настройки ребра — поповер у самого ребра при клике.
3. **Одна акцентная кнопка на экран** — кнопка запуска движка. Всё остальное визуально тихое.
4. **Прогрессивное раскрытие.** Новичок видит только ноды и стрелки. FX, хоткеи, сцены — скрыты до явного запроса.
5. **Мягкий тон.** Строчные буквы, округлые формы. Слова: «start», «live», «add» — не «ENGAGE», «IDLE», «ROUTE».

**Антипримеры:** Voicemeeter, военная панель управления, кокпит самолёта.

## Структура экрана

```
┌─────────────────────────────────────────────┐
│  TOPBAR (единственная постоянная полоса)     │
│  [NODUS logo]  [◀ Scene 1 × ▶] [+]  [⋯] [● Live] │
├─────────────────────────────────────────────┤
│                                             │
│              CANVAS (холст)                 │
│         точечная сетка, виньетка            │
│                                             │
│   [floating: engine status pill — center]  │
│                                             │
│                                    [+ −  ↔] │  ← zoom controls (right-bottom)
│  [+ добавить] (left-bottom)                 │
│       ↑                                     │
│  bottom bar (search + tabs)                 │
└─────────────────────────────────────────────┘
```

### Topbar
- **Слева:** логотип NODUS (компактный знак, не надпись)
- **По центру:** вкладки сцен пилюлями — `◀ Scene 1 ×  Scene 2 × ▶  +`
  - стиль: как на `scene-tab-design.png` — компактно, навигация стрелками
  - максимум 3–5 вкладок видно одновременно, остальные — overflow
- **Справа:** кнопка `⋯` (Settings, Import, Export) + главная акцентная пилюля
  - выключено: `▷ Start` (акцентный цвет, залитая)
  - включено: `● Live` (та же кнопка, мягкая пульс-точка — одна короткая анимация при переключении)
  - **УБИТЬ** слова ENGAGE и IDLE везде

### Canvas
- Фон: `--color-canvas` с точечной сеткой (dot grid, тусклые точки ~1px, шаг ~24px)
- Лёгкая виньетка по краям (radial gradient overlay)
- Нет боковых панелей

### Кнопка Engine (floating, центр-сверху)
- Плавающая пилюля по центру экрана сверху (см. `Engine-start-btn.png`)
- При `paused`: `+ engine paused` — тихая, серая
- При `live`: `● engine live` — акцентный цвет
- Это ДУБЛИРУЕТ кнопку в topbar — оба элемента должны быть синхронизированы

### Zoom controls (правый нижний угол)
- Тихая колонка: `+`, `−`, `⤢` (fit to screen)
- Без фона, просто иконки с hover-эффектом

### Кнопка «+ добавить» (левый нижний угол)
- Стиль: см. `V0/add-device-and-app.png`
- При клике открывается панель снизу-слева (не модалка) с двумя секциями:
  - **Играет сейчас** — список автоопределённых приложений с иконками
  - **Устройства** — реальные аудио-устройства системы
  - Поиск сверху панели
  - Выбрал → нода появилась на холсте → панель закрылась

## Bottom Bar (панель снизу, Library нод)

> Реализует то, что раньше было вкладкой Library. Стиль: `Claude-design/bottom bar.png`.

Это плавающая панель внизу по центру. Содержит:
- **Поле поиска** (`search nodes · type a name to highlight...`)
- **Вкладки-иконки** для разных категорий нод:

| Иконка | Вкладка | Что содержит |
|--------|---------|--------------|
| 🕐 | Recent | Последние использованные ноды |
| ⊞ | All | Все доступные типы нод |
| 🎛️ | FX | Эффекты: Compressor, EQ, Gain, Limiter, Reverb |
| ⚡ | Logic | Логика: Push-to-Talk, Hotkey, Toggle, Timer |
| ✦ | Misc / Other | Остальное: Stream Mix, MIDI, OSC, будущие типы |

- Вкладка Devices (физические устройства) **не входит** в bottom bar — она в панели «+ добавить»
- Справа: кнопка `✦` (открыть/закрыть bottom bar) или аналогичный toggle

## Нода — главный компонент

### Анатомия (сверху вниз)

```
  ● source                     ← ярлык: цветная точка + тип строчными (вне карточки)
┌──────────────────────────────┐  ← карточка, border-radius: --radius-node
│  [icon]  Spotify          ○  │  ← иконка приложения + имя + кнопка mute
│          playing · app       │  ← вторичная строка (тускло)
│  ▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░  │  ← VU-метр (тонкая полоска, цвет типа ноды)
│  🔈 ━━━━●━━━━━━━━━━━━ 55%   │  ← слайдер громкости
●                           ●  │  ← порты (цветные точки на краях)
└──────────────────────────────┘
```

### Ярлык над нодой
- `● тип` — маленький, тусклый, строчными (например: `● source`, `● output`, `● mic`, `● virtual`, `● fx`)
- Точка — цвет соответствующего типа

### Обрамление ноды (frosted glass border)
- Взять из `Lovable/node-design.png`: тонкая рамка с эффектом матового стекла
- Рамка **реагирует на курсор**: при приближении курсора — лёгкое свечение/highlight на ближайшей стороне рамки
  - Далеко: `Lovable/node-reaction-to-a-distant-cursor.png` — рамка тихая
  - Близко v1: `Lovable/node-reaction-to-a-close-cursor-v1.png` — рамка чуть светлее
  - Близко v2: `Lovable/node-reaction-to-a-close-cursor-v2.png` — рамка с акцентом на ближайшей стороне
- Реализация: CSS `background: radial-gradient(circle at var(--mouse-x) var(--mouse-y), ...)` через JS mousemove → CSS custom properties

### Содержимое карточки
- Иконка приложения: реальная иконка (Spotify, Discord, игра) или иконка устройства (наушники, динамик, микрофон)
- Имя: крупно (`--font-size-title`, `--font-weight-medium`)
- Вторичная строка: имя системного устройства или статус (`Galaxy Buds`, `not running`, `virtual`) — тускло (`--color-text-secondary`)
- **VU-метр**: тонкая горизонтальная полоска (~3px), заполняется цветом типа ноды, данные ~15 раз/сек
- **Слайдер громкости**: стандартный range input, кастомный стиль, иконка mute рядом

### Состояния ноды

| Состояние | Визуал |
|-----------|--------|
| Обычная | Стандартный вид, без свечения |
| **Активная** (звук идёт) | Мягкое статичное свечение (`--glow-*`) цветом типа + живой VU-метр |
| **Muted** | Карточка приглушена (opacity 0.6), иконка перечёркнутого динамика |
| **Выделенная** | Тонкая акцентная обводка (`--color-accent`, 1.5px) |
| **Не запущено** | Полупрозрачная (opacity 0.45), серая точка вместо цветной |
| **Компактная** | Уменьшенная версия: только иконка + имя + VU-метр, без слайдера |

> ⚠️ Свечение — статичное, НЕ анимированное. Приложение работает на слабых ноутбуках.

### Порты
- Маленькие цветные круглые точки (~8px) на левом/правом краю карточки
- Цвет = тип ноды
- При перетаскивании нового ребра: пунктирная «резинка» от порта к курсору

## Рёбра (graph lines / wires)

> Стиль: `Claude-design/graph-line-design.png`

- Мягкие кривые Безье (cubic-bezier)
- Толщина: 1.5–2px
- Цвет в покое: тихий серый (~`#3A3A42`)
- **Активное** (звук идёт): чуть ярче (~`#6B6B78`), допустимо слабое свечение
- **Muted**: пунктир (`stroke-dasharray: 6 4`), цвет `--color-muted-edge`
- **НЕТ** постоянных бегущих анимаций (без moving dash-offset)
- При перетаскивании: пунктирная резинка

### Поповер ребра (клик по ребру)
Маленький поповер прямо у ребра, содержит:
- Слайдер громкости
- Кнопка Mute (toggle)
- Баланс L/R (тонкий двойной слайдер или степпер)
- Кнопка удалить ребро (иконка ×, тихая)
- Стиль: `border-radius: --radius-popover`, тёмный фон, тонкая обводка

## Пустой холст (первый запуск)

По центру холста — **призрачная схема-подсказка**:
- Полупрозрачные силуэты нод и стрелки (opacity ~0.2)
- Одна строка под ними: `«перетащи приложение или устройство — или начни с готовой схемы»`
- 2–3 карточки-пресета:
  - `🎮 Стримлю: OBS слышит игру, я — нет`
  - `🎵 Музыка в Discord`
  - `🎧 Всё → наушники`
- Клик по пресету → граф собирается автоматически → призрак исчезает
- Призрак и пресеты исчезают, как только появилась первая нода

## Онбординг-модалка (виртуальный драйвер)

При первом запуске — мягкая модалка в том же стиле:
- Одна фраза объяснения без техжаргона
- Две карточки выбора: `Установить автоматически` / `Позже`
- Кнопка `пропустить` — тихая, второстепенная
- Технические детали (bcdedit, Test Mode, EV cert) — под спойлером «подробнее»

## Что обязательно сохранить (уже работает)

При редизайне каждая функция должна быть достижима — пусть и спрятана:

| Функция | Где в новом UI |
|---------|---------------|
| Список аудио-устройств + rescan | Панель «+ добавить» → Devices |
| Автоопределение приложений | Панель «+ добавить» → Playing now |
| Per-route громкость / mute / баланс L/R | Слайдер на ноде + поповер ребра |
| Solo на ноде | Контекстное меню ноды (ПКМ) |
| VU-метры (~15 fps) | Полоска внутри карточки ноды |
| Сцены + Import/Export | Topbar вкладки + кнопка `⋯` |
| Push-to-Talk | Настройки ноды микрофона (не главный экран) |
| Кнопка запуска движка | Topbar пилюля + плавающий центр-сверху |
| Виртуальные устройства Nodus | Тип `virtual`, фиолетовый цвет |

## Производительность — жёсткое ограничение

- **Запрещены:** `animation: infinite`, `stroke-dashoffset` анимации в покое, постоянные `requestAnimationFrame` без активного звука
- **Разрешены:** короткие CSS transition (hover, появление поповера — до 200ms), VU-метр (обновляется только при активном сигнале), одна короткая анимация при переключении движка
- **Свечение:** статичный `box-shadow`, НЕ анимированный

---

## Язык интерфейса

- Основной: **английский** (текущий)
- Тон: человеческий, строчные буквы, без жаргона
- Примеры: `start`, `live`, `add`, `mute`, `source`, `output` — ОК
- Запрещено: `ENGAGE`, `IDLE`, `ROUTE`, `SPLITTER`, `MIXER`
- Слова «Splitter» и «Mixer» не существуют в интерфейсе — ноды просто имеют много входов/выходов
